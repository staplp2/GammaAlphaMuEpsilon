597/**********************************
598 * Bulk processing dictionary API

601/* A Dictionary is useful for the compression of small messages (KB range).
602 * It dramatically improves compression efficiency.
603 *
604 * LZ4 can ingest any input as dictionary, though only the last 64 KB are useful.
605 * Best results are generally achieved by using Zstandard's Dictionary Builder
606 * to generate a high-quality dictionary from a set of samples.
607 *
608 * Loading a dictionary has a cost, since it involves construction of tables.
609 * The Bulk processing dictionary API makes it possible to share this cost
610 * over an arbitrary number of compression jobs, even concurrently,
611 * markedly improving compression latency for these cases.
612 *
613 * The same dictionary will have to be used on the decompression side
614 * for decoding to be successful.
615 * To help identify the correct dictionary at decoding stage,
616 * the frame header allows optional embedding of a dictID field.
617 */
618typedef struct LZ4F_CDict_s LZ4F_CDict;
